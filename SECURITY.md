# Security Policy

Do not report a security vulnerability in a public GitHub Issue.

Send a minimal description to [support@wetongbu.com](mailto:support@wetongbu.com) with the subject `SECURITY: WeTongbu`. Include the affected version, impact, and reproduction outline, but do not send real credentials, cookies, device tokens, personal note contents, or active signed URLs. We will request additional evidence through a safer channel if needed.

Supported versions are the latest Obsidian community-plugin release and the immediately preceding release when a security update is being rolled out.

## 安全问题

安全漏洞请勿提交公开 Issue。请发送最小必要说明到 [support@wetongbu.com](mailto:support@wetongbu.com)，邮件标题使用 `SECURITY: WeTongbu`。说明受影响版本、影响范围和复现轮廓，但不要发送真实密钥、Cookie、设备令牌、私人笔记或仍有效的签名链接。
