# Support

Choose the channel based on what the report contains:

- Use the [private WeTongbu support form](https://wetongbu.com/support/report/) for account, billing, storage, private-content, or task-specific problems. The form creates a trackable `WTB-...` ticket.
- Use [GitHub Issues](https://github.com/ygx2378/wetongbu-obsidian-plugin/issues) only for reproducible plugin bugs that can be described publicly without private content.
- Use the [setup guide](https://wetongbu.com/docs/quick-start/) for installation and object-storage configuration.

Never publish access keys, secret keys, verification codes, recovery codes, cookies, device tokens, full note contents, or signed URLs. WeTongbu does not need these values to begin triage.

## Receive issue notifications

Repository maintainers can receive new issue emails from GitHub without sharing an inbox password:

1. Open the repository page on GitHub.
2. Select **Watch** → **Custom**.
3. Enable **Issues** and confirm.
4. Verify the GitHub account's notification email and delivery settings.

This is a personal GitHub notification setting; the repository cannot enable it on behalf of another person.
